Three layers.
No center.

Every component is replaceable. Every rule is verifiable. Every decision is on-chain.

Immutable truth
  • DAGKnight consensus · 100 BPS
  • Silverscript smart contracts
  • KRC-20 rule storage (supply = 1)
  • Native Groth16 ZK-verification
  • Sub-second finality
  • ValidatorStaking · RuleStorage

  • Complex governance logic
  • ZK-rollups anchored to L1
  • Reputation computation
  • Reward distribution
  • Dev grant DAO voting
  • Auto-tuning parameters

  • Phi-3-mini local inference

03 / Node types

Four roles.
Every device contributes.

01 · Light Client
Light Client
Your device. Phi-3-mini runs entirely local. Detects anomalies, hashes them, generates ZK proofs, reports anonymously. Zero raw data leaves.
4 GB RAM · No GPU required
Windows · macOS · Linux · ARM
02 · Guardian
Guardian Node
Runs LLaMA 3. Aggregates threat hints, performs deep analysis, generates YARA rules. Earns PROM reputation for accepted proposals.
8B: RTX 4070 Ti+, 16 GB VRAM
70B: 4× A100 / H100
03 · Validator
Validator Node
Stakes KAS. Votes on rule proposals via Commit-Reveal. Provides economic security. Slashed for misbehavior. 7-day exit cooldown.
Minimum 10,000 KAS stake
Standard server · No GPU needed
04 · Honeypot
Honeypot Node
Emulates vulnerable services. Every attack captured and analyzed. First line of zero-day detection. Earns 5% of total PROM emission.
Any internet-exposed server
Higher reward per zero-day find
